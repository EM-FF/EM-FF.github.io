二、Git 分支操作

2.1什么是分支

在版本控制过程中，同时推进多个任务，为每个任务，我们就可以创建每个任务的单独分支。使用分支意味着程序员可以把自己的工作从开发主线上分离开来，开发自己分支的时候，不会影响主线分支的运行。对于初学者而言，分支可以简单理解为副本，一个分支就是一个单独的副本。（分支底层其实也是指针的引用）

2.2分支的操作

git branch 分支名 创建分支
git branch -v 查看分支
git checkout 分支名 切换分支
git merge 分支名 把指定的分支合并到当前分支上

2.3合并分支

  • 基本用法：

在 master 分支上合并 hot-fix 分支

git merge hot-fix
  • 产生冲突

冲突产生的表现：后面状态为 MERGING

86158@LAPTOP-ILBF6UBU MINGW64 /d/code-material/git-space/git-demo (master|MERGING)

冲突产生的原因：
合并分支时，两个分支在同一个文件的同一个位置有两套完全不同的修改。Git 无法替我们决定使用哪一个。必须人为决定新代码内容。

  • 解决冲突

编辑有冲突的文件，删除特殊符号，决定要使用的内容

添加到暂存区

执行提交（注意：此时使用 git commit 命令时不能带文件名）

4.1创建远程仓库

4.2远程仓库操作

git remote -v 查看当前所有远程地址别名
git remote add 别名 远程地址 起别名
git remote remove 仓库别名 删除仓库别名
git remote rename old别名 new别名 给仓库别名起新的名字
git push 别名 分支 推送本地分支上的内容到远程仓库
git pull 远程库地址别名 远程分支名 将远程仓库对于分支最新内容拉下来后与
当前本地分支直接合并
git clone 远程地址 将远程仓库的内容克隆到本地

远程地址：点击code，复制https地址

  • git clone:拉去代码；初始化本地仓库；创建别名（就是将整个仓库cope下来）

  • 要想对clone下来的仓库修改并push到远程仓库，需对方邀请你加入团队（只是修改并不需要）（成员管理在Settings->Collaborators-> Add people）

4.3跨团队协作操作

  • 将远程仓库的地址复制发给邀请跨团队协作的人

  • 受邀人在对方的 GitHub 账号里的地址栏复制收到的链接，然后点击Fork 将项目叉到自己的本地仓库

  • 受邀人修改仓库，最后新建一个pull request

  • 对方仓库收到一个pull request，最后决定是否将修改的代码合并到自己的仓库里

五、IDEA集成git

idea版本2021.3

5.1 配置 Git 忽略文件

在根目录下新建git.ignore文件

# Compiled class file
*.class

# Log file
*.log

# BlueJ files
*.ctxt

# Mobile Tools for Java (J2ME)
.mtj.tmp/

# Package Files #
*.jar
*.war
*.nar
*.ear
*.zip
*.tar.gz
*.rar


hs_err_pid*

.classpath
.project
.settings
target
.idea
*.iml

再在.gitconfig文件中添加git.ignore的路径

5.2导入git

  • Settings->Version Control->Git

  • 选择安装的git.exe（在bin目录下）

5.3初始化本地库

选择左上角的VSC，选择Create Git Repository,选择该项目

5.4提交、添加

右键项目目录 Git->add 、 Git->commit

未添加的文件，文件名是红色的，添加过后变蓝色

提交过后，文件名恢复成平时的白色

5.5切换版本

  • 在 IDEA 的左下角，点击 Version Control，然后点击 Log 查看版本

  • 右键选择要切换的版本，然后在菜单里点击 Checkout Revision

5.6创建分支

  • 选择 Git，在 Repository 里面，点击 Branches 按钮

  • 在弹出的 Git Branches 框里，点击 New Branch 按钮

  • 填写分支名称，创建 分支。

  • 右下角查看当前在哪个分支下

  • 快捷方法：也可以直接点击右下角分支，直接创建分支

5.7切换分支

点击在右下角分支图标，选中分支，点击checkout

5.8合并分支

点击右下角分支图标，选中要合并的分支，注意当前是你的主分支

5.9解决合并冲突

如果 master 分支和 hot-fix 分支都修改了代码，在合并分支的时候就会发生冲突

点击 Conflicts 框里的 Merge 按钮，进行手动合并代码

》表示将分支新增加代码加入到原版本中，×表示丢弃

手动合并完代码以后，点击右下角的 Apply 按钮

代码冲突解决，自动提交本地库

六、Idea集成Github

6.1 设置 GitHub 账号

Settings(右上角的小齿轮/黄色↑) —> Version Control

6.2分享项目到Github

左上角的Git –> GitHub –> share project on github

6.3 push 推送本地库到远程库

Git –> Push

注意：push 是将本地库代码推送到远程库，如果本地库代码跟远程库代码版本不一致，
push 的操作是会被拒绝的。也就是说，要想 push 成功，一定要保证本地库的版本要比远程
库的版本高！因此一个成熟的程序员在动手改本地代码之前，一定会先检查下远程库跟本地
代码的区别！如果本地的代码版本已经落后，切记要先 pull 拉取一下远程库的代码，将本地
代码更新到最新以后，然后再修改，提交，推送！

6.4 pull 拉取远程库到本地库

Git -> Pull

注意：pull 是拉取远端仓库代码到本地，如果远程库代码和本地库代码不一致，会自动
合并，如果自动合并失败，还会涉及到手动解决冲突的问题

6.5 clone 克隆远程库到本地

初始化界面 选择VSC，用Repository url，复制仓库的ssh或https
