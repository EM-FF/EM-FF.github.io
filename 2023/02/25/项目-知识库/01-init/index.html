技术选型

前端：前端3件套html、css、JavaScript, vue3, Ant Design Vue
后端：

  • java(8.0)
  • spring(DI更好管理项目依赖)
  • springmvc(restful编程风格，提供接口访问)
  • mybatis(java操作数据库的框架)
  • mybatis-plus(mybatis加强版，不用写sql语句就可以增删改查)
  • springboot(快速启动项目，集成框架 2.7.10 稳定版本)
  • junit(单元测试)
  • mysql(关系型数据库，使用8.x版本)

第一天

初始化项目，使用git将代码提交到本地仓库，并提交到gitee(在家里提交到github超时，改用github)
controller接口测试 TestController

日志优化

logback-spring.xml 日志配置文件

restful风格

controller层接口测试

使用以.http结尾文件测试
例：test.http 用于TestController中/hello接口测试

热部署

springboot 2.7.10 idea2021.3.3

  1. 添加依赖 2.添加配置 3.测试

配置文件

在resource中添加config文件夹，新建一份application.yml配置文件用于项目上线使用；在resource下的application.yml用于生产测试使用。
当两个配置文件有配置冲突时，config下的配置文件生效

解决配置错误小技巧

项目出现配置问题，关闭项目，在文件资源管理器中将.idea文件夹删除，然后重新打开项目
.idea放置项目配置

git

git init 初始化仓库
git branch -M main 选择分支
git remote add origin https:github.com/xxx 给远程仓库起个别名
git add 将新文件提交到暂存区 (git add . 提交所有新文件) idea中文件绿色表示已添加到暂存区，红色未添加
git commit 将暂存区中文件提交到本地仓库 可以使用idea右上角的 绿色√ 提交代码到本地仓库
git push 提交到远程仓库 可以idea使用右上角 斜右上方箭头 将代码push到远程仓库
